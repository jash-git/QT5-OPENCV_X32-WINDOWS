win10下QT5配置OpenCV方法

 2020-05-30 
opencvpenqt5

作為一個QT小白，最近在配置QT5+OpenCV,卻發現總是踩坑。好在我成功配置好了。

系統版本：win10 64位系統

QT5版本：qt-opensource-windows-x86-5.12.3.exe

Cmake版本：cmake-3.17.0-win64-x64.zip

OpenCV版本：opencv-4.3.0.zip

首先安裝QT

win10中安裝軟件最好不要裝在C盤，避免權限問題。

組件中一定要選中MinGW版本的編譯器，我這裡選擇32位，QT其他的請隨意

還有Developer and Designer Tools中，也要選中MinGW

然後下一步，等待QT安裝成功。

安裝Cmake

解壓cmake 3.17，

然後下一步更改安裝路徑到D盤，繼續點擊直至安裝完成。

設置系統環境變量

將如下目錄添加到系統環境變量中（QT安裝目錄和CMake安裝目錄）：

D:\Qt\Qt5.12.3\Tools\mingw730_32\bin

D:\Qt\Qt5.12.3\5.12.3\mingw73_32\bin

D:\CMake\bin

重啟一下系統。

配置Cmake

解壓OpenCV到D盤。

在D盤建立一個任意英文目錄，我取名為opencv，再打開Cmake

where id the source code:這裡選擇opencv的源文件目錄

where to build the binaries:這裡選擇後面新建的那個目錄

勾選Advanced,點擊Configure

點擊下一步,後稍作暫停。

打開QT，選擇工具→選項→編譯器，查看C和C++的路徑

把gcc和g++的路徑複製到cmake，點擊完成，後等待

勾選中“WITH_OPENGL”、“WITH_QT”；不勾選“WITH_IPP”。

特別的，不要勾選如下選項

否則會出現

modules\core\CMakeFiles\opencv_core.dir\build.make:1506:modules/core/CMakeFiles/opencv_core.dir/vs_version.rc.obj

錯誤，需要手動編譯vs_version.rc.obj文件。

然後再次點Configure。

出現如下錯誤，檢查路徑是否正確

確認後，再次點Configure，直到沒有錯誤為止。

點擊Generate，稍等片刻。

編譯OpenCV

打開windows powershell，定位到自己後面新建的目錄。我這裡是opencv，執行mingw32-make

該命令可以調用多線程執行：mingw32-make -j8；這表示使用8線程

完成之後輸入mingw32-make install

將以下路徑添加到系統環境變量

D:\opencv\install\x64\mingw\bin

D:\opencv\bin

重啟電腦。

測試OpenCV

在QT的pro文件中添加以下代碼：

INCLUDEPATH += D:\opencv\install\include

建立QT工程進行測試
